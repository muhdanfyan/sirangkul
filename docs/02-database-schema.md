# Skema Database SiRangkul

Dokumen ini menjelaskan struktur data dan relasi dari database SiRangkul. Skema lengkap bisa dilihat pada file migrasi Laravel di `database/migrations/`.

## 1. Tabel Inti

### `users`
Tabel pengguna untuk autentikasi dan manajemen hak akses.
- **id**: UUID Utama.
- **full_name**: Nama lengkap pengguna.
- **email**: Email unik untuk autentikasi portal.
- **password**: Hash Bcrypt.
- **role**: Hak Akses (ENUM: `administrator`, `pengusul`, `verifikator`, `kepala_madrasah`, `bendahara`, `komite_madrasah`).
- **status**: Status akun (`Active` | `Inactive`).

### `rkam` (Rencana Kerja dan Anggaran Madrasah)
Kumpulan pos anggaran tahunan/tahap yang menjadi pagu dana untuk proposal.
- **id**: UUID Utama.
- **kode_rkam**: Identitas unik pos anggaran.
- **nama_program**: Penjelasan terkait program/pos dana.
- **kategori**: Jenis kategori anggaran (ENUM: `Operasional`, `Pembinaan`, `Infrastruktur`, `Lainnya`).
- **pagu_anggaran**: Total dana yang dialokasikan (DECIMAL).
- **realisasi**: Total dana yang sudah disetujui untuk pos ini (DECIMAL). Default: 0.
- **sisa_anggaran**: Selisih otomatis dari `pagu_anggaran` - `realisasi`.
- **tahun_anggaran**: Parameter periode aktif untuk data ini.
- **status**: Ketersediaan pos (ENUM: `Aktif`, `Selesai`, `Draft`).
- **created_by** / **updated_by**: Audit pembuat (FK -> `users`).

### `proposals`
Tabel transaksi utama pengajuan pencairan dana/kegiatan.
- **id**: UUID Utama.
- **pengusul_id**: Pemilik proposal (FK -> `users`).
- **rkam_id**: Target penyedot dana (FK -> `rkam`). Nullable jika sedang draf awal.
- **judul_proposal**: Judul proposal.
- **kode_proposal**: Autogenerated sequential ref.
- **latar_belakang**, **tujuan_kegiatan**, **deskripsi_kegiatan**, **indikator_keberhasilan**: Narasi kegiatan.
- **tanggal_mulai**, **tanggal_selesai**: Waktu pelaksanaan.
- **kategori**: (ENUM: `Sains_Teknologi`, `Olahraga_Seni`, `Keagamaan`, `Operasional_Fasilitas`, `Lainnya`).
- **total_anggaran**: Jumlah uang yang diajukan (DECIMAL).
- **status**: Status lifecycle (ENUM: `Draft`, `Submitted`, `Verified`, `Approved`, `Final_Approved`, `Rejected`, `Revision`, `Funded`, `Completed`).
- **tanggal_pengajuan**, **verified_at**, **approved_at**, **final_approved_at**: Audit waktu tahapan.

### `proposal_attachments`
File-file pendukung (softcopy) pada setiap proposal.
- **proposal_id**: FK relasi.
- **file_name**: Nama asli file.
- **file_path**: Path pada storage lokal atau cloud (S3).
- **file_type**: Format MIME type atau klasifikasi.
- **file_size**: Ukuran (opsional untuk frontend).

### `approval_workflows`
Tabel pencatatan riwayat persetujuan atau penolakan. Mengandung kronologi dokumen.
- **id**: UUID Utama.
- **proposal_id**: Target catatan (FK -> `proposals`).
- **user_id**: Petugas yang mengambil keputusan (Verifikator / Kepala Madrasah / Komite).
- **action**: Keputusan persetujuan (`Verified`, `Approved`, `Rejected`, `Returned`).
- **notes**: Alasan yang ditulis pengguna saat mereka menyetujui atau menolak.

### `payments`
Tabel pencatatan pencairan/keuangan oleh bendahara.
- **proposal_id**: Proposal induk (FK -> `proposals`).
- **bendahara_id**: Identitas bendahara pembayar (FK -> `users`).
- **jumlah_dibayar**: Nominal akhir pencairan.
- **metode_pembayaran**: Opsional (Tunai/Transfer).
- **bukti_transaksi_path**: Kwitansi, Struk, Berita Acara (storage URL).
- **catatan**: Info opsional pencairan.
- **status_pembayaran**: (`Pending`, `Completed`, `Failed`, `Cancelled`).

## 2. Tabel Ekstra
- `notifications`: Sistem pesan in-app untuk mengabari pengusul, verifikator dll.
- `audit_logs`: Tabel keamanan super ketat. Semua C-R-U-D penting (terutama action dari admin), dicatat ke dalam log history untuk transparansi (Siapa, Mengubah apa, Kapan).
- `feedback`: Formulir keluhan atau test feedback antar pengguna sistem.

## Catatan Relasi
- Relasi **Cascade On Delete** wajib dicegah (Restrict) pada tabel-tabel transaksional (`proposals`, `rkam`, `payments`) demi menjaga Audit Trail pembukuan pada pelaporan tahunan.
